# 訪看かんたん売上管理

5施設の訪問看護実績を週に1回入力し、売上見込みと月間目標を確認するためのWindowsデスクトップアプリです。

## フェーズ0の範囲

- Electron + Vue 3 + TypeScript + Vite の最小構成
- Main / Preload / Renderer の分離
- Rendererから型付きAPIを呼ぶIPC疎通
- Vitest、ESLint、Prettier、electron-builder の導入
- Windows用NSISインストーラー生成

SQLite、施設CRUD、初期設定ウィザードはフェーズ1で実装します。

## 開発コマンド

PowerShellでは `npm.ps1` が実行ポリシーで止まる場合があるため、`npm.cmd` を使います。

```powershell
npm.cmd install
npm.cmd run dev
npm.cmd run test
npm.cmd run typecheck
npm.cmd run build
npm.cmd run dist
```

## フェーズ0 手動確認

1. `npm.cmd run dev` を実行する。
2. Electronウィンドウが開くことを確認する。
3. ホーム画面に「アプリの準備ができています」と表示されることを確認する。
4. IPC確認欄に「Main / Preload / Renderer の接続を確認しました。」と表示されることを確認する。
5. `npm.cmd run dist` を実行し、`release` 配下にWindows用セットアップファイルが生成されることを確認する。

## フェーズ1 手動確認

1. `npm.cmd run dev` を実行する。
2. 初回起動で「初期設定」画面が表示されることを確認する。
3. 5施設名と、介護・別表7・精神・特指示それぞれの単価を入力する。
4. 「初期設定を完了する」を押し、ホーム画面へ移動することを確認する。
5. アプリを再起動し、初期設定画面が再表示されないことを確認する。
6. 「目標・単価設定」で施設名を変更し、再起動後も施設名が残ることを確認する。

## フェーズ2 手動確認

1. `npm.cmd run dev` を実行する。
2. 「目標・単価設定」を開き、「売上単価」タブで適用開始月を選ぶ。
3. 介護・別表7・精神・特指示の単価を千円単位で入力し、保存できることを確認する。
4. 「月間目標」タブで対象年月を選び、施設・訪問看護区分ごとの目標人数、目標売上を保存する。
5. 前月に目標を保存した後、翌月で「前月の目標をコピー」を押し、同じ内容がコピーされることを確認する。
6. アプリを再起動し、単価と月間目標が残ることを確認する。

## フェーズ3 手動確認

1. `npm.cmd run dev` を実行する。
2. 「週次入力」を開き、対象年月に `2026年7月` を選ぶ。
3. 7月1日（水）〜7月4日（土）から始まる5期間が表示されることを確認する。
4. 対象年月に `2026年8月` を選び、8月1日（土）だけの期間を含む6期間が表示されることを確認する。
5. すべての期間に曜日と日数が表示され、月外の日付が含まれないことを確認する。

## フェーズ4 手動確認

1. `npm.cmd run dev` を実行する。
2. 「目標・単価設定」で対象月の単価を登録する。
3. 「週次入力」を開き、対象年月、期間、施設を選べることを確認する。
4. 5施設を施設ボタンで切り替えながら人数を入力し、「保存して次の施設へ」で連続入力できることを確認する。
5. 入力直後に人数合計、売上見込みが更新されることを確認する。
6. 「一時保存」後にアプリを再起動し、同じ期間・施設の入力が残っていることを確認する。
7. 空欄を0以上の整数で埋めて「入力完了」を押し、保存状態が「入力完了」になることを確認する。
8. 空欄がある状態では「入力完了」にできず、日本語の案内が表示されることを確認する。

## フェーズ5 手動確認

1. `npm.cmd run dev` を実行する。
2. 「週次入力」を開き、対象月の施設×期間マトリクスが表示されることを確認する。
3. 未入力セルを押すと、同じ期間・施設の入力欄へ移動することを確認する。
4. 一時保存と入力完了を行い、マトリクスの表示が「一時保存」「入力完了」に変わることを確認する。
5. 第1期間を保存した後、第2期間で「前期間をコピー」を押し、人数がコピーされて一時保存になることを確認する。
6. 第1期間では「前期間をコピー」が押せないことを確認する。
7. 月初・月末など日数が違う期間へのコピー時に、日数差の確認文が表示されることを確認する。

## フェーズ6 手動確認

1. `npm.cmd run dev` を実行する。
2. ホーム画面で対象年月、前月、今月、翌月を切り替えられることを確認する。
3. 月間目標、月間累計、達成率、目標までまたは目標超過、月末予測が千円単位で表示されることを確認する。
4. 週次入力で一時保存した内容が月間累計に含まれることを確認する。
5. 入力完了した期間だけが月末予測の完了日数に含まれることを確認する。
6. 施設別、看護区分別、期間別推移の表が表示されることを確認する。
7. 目標0円の月でも達成率が安全に `－` 表示になることを確認する。

## フェーズ7 手動確認

1. `npm.cmd run dev` を実行する。
2. 「過去月」を開き、対象年月を切り替えられることを確認する。
3. 未入力がある月では、月締めボタンが押せないことを確認する。
4. 全施設・全期間を入力完了にした月で、「この月を締める」を押せることを確認する。
5. 締め済み月で週次入力や月間目標保存ができないことを確認する。
6. 「この月を再開する」を押すと、再び入力・目標保存できることを確認する。
7. 前月との差額が表示されることを確認する。

## フェーズ8 手動確認

1. `npm.cmd run dev` を実行する。
2. 「バックアップ・設定」を開き、対象年月を選ぶ。
3. 詳細CSVと月間集計CSVを保存し、Excelで日本語が読めることを確認する。
4. 「データをバックアップ」で `.db` ファイルを保存できることを確認する。
5. データを変更した後、バックアップから復元して元の内容に戻ることを確認する。
6. 復元前に自動退避バックアップが作られることを確認する。
7. 壊れたDBファイルを復元しようとしても、現在のデータが維持されることを確認する。

## データ保存方針

フェーズ1以降、データベースはElectronの `userData` 配下に作成します。インストール先フォルダーには保存しません。

## 配布時の注意

このアプリはMVPではコード署名なしでもビルドできる設定です。署名なしのSetup.exeを配布した場合、Windows SmartScreenやウイルス対策ソフトが警告を表示することがあります。

上書きインストールでは、アプリ本体だけを更新し、入力済みデータはElectronの `userData` 配下に残す方針です。データベースはインストール先フォルダーではなく、Windowsユーザーごとのアプリデータ領域に保存されます。

通常のアンインストールでは、インストールされたアプリ本体は削除されますが、`userData` 配下のデータベースは自動削除しません。入力済みデータを完全に削除したい場合は、バックアップを確認したうえでユーザーのアプリデータ領域から削除してください。
