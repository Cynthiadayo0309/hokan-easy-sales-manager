# 訪看かんたん売上管理

5施設の訪問看護実績を週に1回入力し、売上見込みと月間目標を確認するためのWindowsデスクトップアプリです。

## 開発背景（STAR）

### Situation：どんな状況だったのか

担当する5施設の訪問看護実績について、会社が設定した月間目標と、日々の実績から見込まれる売上を比較する必要がありました。実績は医療・精神科・介護などの区分に分かれ、さらに1人あたりの訪問回数によって売上単価が異なります。そのため、手作業での計算や集計は作業負担が大きく、月をまたぐ週の集計や転記時にミスが起きるリスクもありました。

また、主な利用者はパソコン操作に不慣れであり、複雑な表計算やコマンド操作を前提にできませんでした。インターネット接続に依存せず、Windowsパソコン1台で簡単かつ安全に運用できる仕組みが求められていました。

### Task：どのような課題設定をしたのか

「利用者が週に1回、各施設の人数を入力するだけで、売上見込みと月間目標の進捗を正確に把握できること」を中心課題に設定しました。具体的には、次の状態を目指しました。

- 5施設分の入力を迷わず連続して行える
- 訪問人数から訪問件数と売上見込みを自動計算できる
- 月初・月末を含む期間を正しく分割し、対象月の実績だけを集計できる
- 概算売上・確定売上・月間目標・達成率・差額をひと目で比較できる
- 入力漏れや誤操作を防ぎ、入力済みデータを安全に保持できる
- 専門知識やインターネット接続なしで利用を開始できる

### Action：課題に対して、どんな対策を取ったのか

入力作業を「対象月・期間・施設を選び、人数を入力する」という流れに絞り、人数から件数と売上を自動計算するようにしました。「保存して次の施設へ」「前期間をコピー」「すべて0件にする」「一時保存」などを用意し、5施設を繰り返し入力する負担を減らしています。

集計の正確性を保つため、週を日曜日から土曜日で扱いつつ、月初・月末では対象月に含まれる日付だけを期間として生成しています。また、未入力の可視化、入力完了時の検証、月締め後の編集制限、重要操作の確認画面を設け、入力漏れや意図しない変更を防ぐ設計にしました。

結果の確認画面では、施設別・看護区分別・期間別の実績に加え、概算売上、確定売上、達成率、目標との差額、月末予測をまとめて表示します。アプリは Electron・Vue 3・TypeScript・SQLite でWindows向けのデスクトップアプリとして実装し、オフライン動作、ローカル保存、CSV出力、データベースのバックアップ・復元にも対応しました。

### Result：その結果、どんな成果を上げたのか

週次の人数入力から、5施設分の売上計算、月間集計、目標との比較までを一つのアプリ内で完結できるようになりました。これにより、訪問回数区分ごとの単価計算や施設別集計を手作業で行う必要がなくなり、利用者は入力直後に最新の売上見込みと進捗を確認できます。

さらに、未入力箇所の表示や月締め機能によって確認作業を標準化し、CSV出力とバックアップ・復元によって報告データの活用とデータ保全も行えるようになりました。定量的な業務削減効果は今後の運用で計測する必要がありますが、少なくとも「複雑な計算をせず、人数入力だけで判断に必要な情報を得る」という当初の課題を満たす運用基盤を実現しました。

## フェーズ0の範囲

- Electron + Vue 3 + TypeScript + Vite の最小構成
- Main / Preload / Renderer の分離
- Rendererから型付きAPIを呼ぶIPC疎通
- Vitest、ESLint、Prettier、electron-builder の導入
- Windows用NSISインストーラー生成

SQLite、施設CRUD、初期設定ウィザードはフェーズ1で実装します。

## 開発コマンド

PowerShellでは `npm.ps1` が実行ポリシーで止まる場合があるため、`npm.cmd` を使います。

```powershell
npm.cmd install
npm.cmd run dev
npm.cmd run test
npm.cmd run typecheck
npm.cmd run build
npm.cmd run dist
```

## データ保存方針

フェーズ1以降、データベースはElectronの `userData` 配下に作成します。インストール先フォルダーには保存しません。

## 配布時の注意

このアプリはMVPではコード署名なしでもビルドできる設定です。署名なしのSetup.exeを配布した場合、Windows SmartScreenやウイルス対策ソフトが警告を表示することがあります。

上書きインストールでは、アプリ本体だけを更新し、入力済みデータはElectronの `userData` 配下に残す方針です。データベースはインストール先フォルダーではなく、Windowsユーザーごとのアプリデータ領域に保存されます。

通常のアンインストールでは、インストールされたアプリ本体は削除されますが、`userData` 配下のデータベースは自動削除しません。入力済みデータを完全に削除したい場合は、バックアップを確認したうえでユーザーのアプリデータ領域から削除してください。
