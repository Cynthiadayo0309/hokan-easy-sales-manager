# 06. テスト・受入条件

## 1. テスト方針

特に次の領域を自動テストする。

- 日付期間生成
- 人数・件数計算
- 売上計算
- 千円と円の変換
- 達成率
- 月次確定売上
- 月全体売上目標
- 目標差額
- 月末予測
- DBマイグレーション
- バックアップ・復元

## 2. 期間生成テスト

### ケース1: 2026年7月

期待値:

1. 7/1～7/4
2. 7/5～7/11
3. 7/12～7/18
4. 7/19～7/25
5. 7/26～7/31

### ケース2: 2026年8月

期待値:

1. 8/1
2. 8/2～8/8
3. 8/9～8/15
4. 8/16～8/22
5. 8/23～8/29
6. 8/30～8/31

### ケース3: 月初が日曜日

第1期間が7日間になること。

### ケース4: 月末が土曜日

最終期間が土曜日で終了し、翌月の日付を含まないこと。

### 共通条件

- 期間に重複がない
- 期間に欠落がない
- 月初から月末をすべてカバーする
- 日付が対象月外へ出ない

## 3. 件数計算テスト

入力:

- 1回: 10人
- 2回: 3人
- 3回: 2人

期待値:

- 人数: 15人
- 件数: 22件

## 4. 売上計算テスト

入力:

- 1回: 10人、単価8,500円
- 2回: 3人、単価16,000円
- 3回: 2人、単価23,000円

期待値:

- 売上: 179,000円
- 表示: 179千円

2回単価へ2を、3回単価へ3を追加で掛けていないこと。

## 5. 千円変換テスト

|      入力 |    内部円額 |      表示 |
| --------: | ----------: | --------: |
|   8.5千円 |     8,500円 |   8.5千円 |
|   500千円 |   500,000円 |   500千円 |
| 1,250千円 | 1,250,000円 | 1,250千円 |

浮動小数誤差による1円差が出ないこと。

## 6. 達成率テスト

月全体の達成率は確定売上を実績として計算する。

### 通常

- 目標: 1,000,000円
- 実績: 750,000円
- 達成率: 75.0%
- 残額: 250,000円

### 超過

- 目標: 1,000,000円
- 実績: 1,080,000円
- 達成率: 108.0%
- 超過額: 80,000円

### 目標0

- 0除算しない
- 達成率を `－` とする
- 実績がある場合は目標未設定を表示する

### 確定売上

- `8.5千円`を8,500円で保存できる
- 0円と未入力を区別できる
- 空欄保存で未入力へ戻せる
- 確定売上を保存しても施設別・看護区分別の概算売上は変わらない
- 締め済み月は保存できない

### 月全体売上目標

- `8.5千円`を8,500円で保存できる
- 0円と未入力を区別できる
- 月全体目標が内訳目標合計より優先される
- 月全体目標未入力時は内訳目標合計へフォールバックする
- 月全体目標を変更しても施設別・看護区分別目標は変化しない

## 7. 月末予測テスト

入力:

- 累計: 600,000円
- 完了日数: 18日
- 月日数: 31日

期待値:

- 約1,033,333円
- 主要表示: 1,033千円程度

完了日数0の場合は予測を表示しない。

## 8. 入力状態テスト

- 新規状態は未入力
- 一部入力・自動保存後は下書き
- コピー後は下書き
- 完了操作後は入力完了
- 締め済み月は編集不可
- 再開後は編集可能
- 確定売上未入力は警告し、確認後は月締め可能
- 月全体目標未入力は警告し、確認後は月締め可能

## 9. 前期間コピーテスト

- 同施設の前期間から人数をコピーする
- 元データが変更されない
- コピー先単価が使用される
- コピー先はdraft
- 日数差がある場合に警告する
- 前期間がない場合はコピー不可と案内する

## 10. DBテスト

- 初回起動でDB作成
- 再起動でデータ保持
- 外部キー制約が有効
- 一意制約が有効
- マイグレーション再実行で重複適用しない
- マイグレーション失敗でロールバックする

## 11. バックアップ・復元テスト

- バックアップファイルを作成できる
- バックアップから同じデータを復元できる
- 壊れたファイルを拒否する
- 復元前に現DBを退避する
- 復元失敗時に現データを維持する
- 旧バージョンDBへマイグレーションを適用できる

## 12. インストーラーテスト

クリーンなWindows環境で確認する。

- Node.jsなしでインストールできる
- SQLiteなしで起動できる
- デスクトップショートカットが作られる
- 初回起動できる
- DBがuserDataに作られる
- 上書きインストールでDBが残る
- アンインストールできる
- オフラインで主要機能が動く

## 13. UI受入テスト

機械操作に不慣れな利用者を想定し、次を確認する。

- 起動後に何をすればよいか分かる
- 対象年月が分かる
- 対象期間が日付で分かる
- 対象施設が分かる
- 人数を入力する場所が分かる
- 保存できたことが分かる
- 次の施設へ進める
- 未入力施設が分かる
- 金額が千円単位だと分かる
- 目標と実績が同じ画面で比較できる
- 概算売上と確定売上を同じ画面で比較できる
- 月全体目標を概算売上・確定売上と同じ画面で入力できる
- 確定売上による達成率が分かる
- エラー文が日本語で理解できる

## 14. 主要受入シナリオ

### シナリオA: 初回利用

1. インストーラーを実行する
2. デスクトップアイコンから起動する
3. 施設名を設定する
4. 単価を設定する
5. 当月目標を設定する
6. ホームを表示する

期待結果: 技術設定なしで利用を開始できる。

### シナリオB: 毎週の入力

1. 今月を選ぶ
2. 対象期間を選ぶ
3. 施設Aの人数を入力する
4. 保存して次の施設へ進む
5. 施設Eまで入力する
6. 入力完了にする
7. ホームで月間累計を見る

期待結果: 5施設を迷わず連続入力できる。

### シナリオC: 月をまたぐ週

1. 7月最終期間を開く
2. 7月26日～7月31日と表示される
3. 7月分だけ入力する
4. 8月を開く
5. 8月1日の期間が表示される

期待結果: 7月と8月が混ざらない。

### シナリオD: バックアップ

1. バックアップを作成する
2. データを変更する
3. バックアップから復元する

期待結果: バックアップ時点の内容へ戻る。
