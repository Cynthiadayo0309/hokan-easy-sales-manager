# 05. 実装ロードマップ

## 1. 基本ルール

- 各フェーズを小さく実装する
- 各フェーズでテストを追加する
- 既存機能を壊さず進める
- DB変更はマイグレーションで行う
- 一時的なモック値を本番コードへ残さない
- フェーズ完了時に手動確認手順をREADMEへ追記する

## フェーズ0: プロジェクト初期化

### 実装

- Electron + Vue 3 + TypeScript + Viteの構成
- ESLint、Prettier
- Vitest
- electron-builder
- Main、Preload、Rendererの分離
- 基本的なIPC疎通
- Windows開発ビルド

### 完了条件

- 開発モードで起動する
- Rendererから型付きAPIを呼び出せる
- Windows用の空インストーラーを生成できる

## フェーズ1: DB・初期設定

### 実装

- SQLite接続
- マイグレーション基盤
- 初期マスター投入
- 施設CRUD
- 初期設定ウィザード
- アプリ設定保存

### 完了条件

- 初回起動でDBが自動作成される
- 5施設名を設定できる
- 再起動後も施設名が残る
- 初回設定完了後はウィザードが再表示されない

## フェーズ2: 単価・月間目標

### 実装

- 売上単価設定
- 適用開始日
- 月間目標設定
- 前月設定コピー
- 千円入力と円保存の変換

### 完了条件

- 3看護区分×3訪問回数の単価を設定できる
- 対象年月・施設・看護区分ごとに目標を保存できる
- 金額に千円単位が明記される
- 8.5千円を8,500円として保存できる

## フェーズ3: 月内期間生成

### 実装

- 日曜始まりの期間生成関数
- 月初・月末の部分期間
- 4～6期間への対応
- 期間一覧UI

### 完了条件

- 2026年7月が5期間になる
- 2026年8月が6期間になる
- 期間が月外の日付を含まない
- 曜日付きで表示される

## フェーズ4: 週次実績入力

### 実装

- 対象月・期間・施設の選択
- 3看護区分×3回数区分の人数入力
- 自動計算
- draft保存
- completed操作
- 施設切り替え
- 保存して次の施設へ
- すべて0件

### 完了条件

- 5施設を連続入力できる
- 人数、件数、売上が即時更新される
- 再起動後も入力内容が残る
- 未入力、下書き、完了を区別できる

## フェーズ5: コピー・入力状況

### 実装

- 前期間コピー
- 日数差警告
- 月間入力状況マトリクス
- 未入力から入力画面への遷移

### 完了条件

- 同じ施設の直前期間をコピーできる
- コピー後はdraftになる
- 日数が違う場合に警告される
- 施設×期間の入力状態が一覧で分かる

## フェーズ6: 月間ダッシュボード

### 実装

- 全体集計
- 施設別集計
- 看護区分別集計
- 達成率
- 目標差額・超過額
- 月末予測
- 期間別推移
- 月全体の確定売上入力
- 月全体の売上目標入力
- 概算売上と確定売上の並列表示
- 確定売上による正式な達成率

### 完了条件

- 目標と累計を同じ画面で比較できる
- 達成率100%超を表示できる
- 目標0の場合に安全に表示できる
- 確定売上未入力と0円を区別できる
- 月全体目標を内訳目標合計より優先できる
- 未完了期間を予測日数に含めない

## フェーズ7: 過去月・月締め

### 実装

- 過去月切り替え
- 前月比較
- 月締め
- 再開
- 締め済み月の読み取り専用化

### 完了条件

- 過去月を閲覧できる
- 未入力がある場合に警告される
- 締め後は通常編集できない
- 管理者操作で再開できる

## フェーズ8: CSV・バックアップ

### 実装

- 詳細CSV
- 月間集計CSV
- DBバックアップ
- DB復元
- 復元前の自動退避
- integrity check

### 完了条件

- Excelで日本語CSVを開ける
- バックアップからデータを復元できる
- 不正ファイルで既存DBが壊れない
- 復元失敗時に元へ戻る

## フェーズ9: UI改善

### 実装

- 大きな文字・ボタン
- 空データ案内
- 日本語エラー
- キーボード操作
- フォーカス表示
- 保存状態表示
- ローディング・二重送信防止

### 完了条件

- 主要操作が迷わず行える
- 技術エラーが画面に直接出ない
- 保存中に二重操作されない
- 色以外でも状態が分かる

## フェーズ10: 配布

### 実装

- NSIS設定
- デスクトップショートカット
- アプリアイコン
- バージョン表示
- userData保持確認
- 上書きインストール確認

### 完了条件

- Windows用Setup.exeを生成できる
- Node.jsなしの端末へインストールできる
- 初回起動できる
- 上書きインストール後もデータが残る
- アンインストール後のデータ扱いをREADMEへ明記する
