# 01. プロダクト要件定義

## 1. プロダクト名

**訪看かんたん売上管理**

### サブタイトル

週1回の入力で、5施設の目標と売上見込みをかんたん確認

## 2. 背景

会社が設定した月間目標と、訪問看護の実績から算出した売上見込みを比較したい。

現在は担当する5施設について、医療訪問看護、精神科訪問看護、介護訪問看護の実績を管理する必要がある。訪問回数によって売上単価が異なり、手作業で計算・集計すると負担が大きい。

利用者は機械操作に不慣れなため、複雑な表計算やコマンド操作を必要とせず、人数を入力するだけで結果を確認できるアプリが必要である。

## 3. 目的

- 5施設の訪問実績を週に1回まとめて入力する
- 訪問人数から訪問件数と売上見込みを自動計算する
- 月内実績だけを正確に集計する
- 会社が設定した月間目標との比較を行う
- 施設ごとの確定売上を入力する
- 確定売上による目標達成率と目標までの差額をひと目で確認する
- 操作に不慣れな利用者でも迷わず使えるようにする
- インストーラーを実行するだけでローカル端末上で利用できるようにする

## 4. 利用者

### 初期バージョン

- 利用者: 管理者1名
- 利用端末: Windowsパソコン1台
- ログイン: 不要
- データ保存: ローカル端末
- インターネット接続: 不要

### 将来

- 管理者と一般利用者の複数人利用
- 利用者ごとの担当施設
- 権限管理
- 複数端末間でのデータ共有

初期バージョンでは複数人利用を実装しないが、将来拡張を妨げない設計とする。

## 5. 管理対象

### 施設

初期状態では5施設を管理する。

- 施設A
- 施設B
- 施設C
- 施設D
- 施設E

施設名は設定画面で変更できる。将来的に施設数を増減できるデータ構造にする。

### 訪問看護の種類

- 医療訪問看護
- 精神科訪問看護
- 介護訪問看護

### 訪問回数区分

1人・1日あたりの訪問回数を次の3区分で管理する。

- 1回訪問
- 2回訪問
- 3回訪問

各区分に、1人あたりの売上単価を設定する。

## 6. 目標設定

会社が決定した目標を対象年月ごとに登録する。

- 施設ごとの月間売上目標
- 施設・訪問看護区分ごとの内訳目標

- 月間目標人数
- 月間目標件数
- 月間目標売上

施設別の売上目標は月次入力画面で編集し、内訳目標は設定画面で編集する。施設別目標が未入力の場合は、その施設の内訳目標合計を使用する。既存の月全体目標は自動配賦せず、施設別入力開始前の参考・互換値として保持する。

## 7. 金額単位

利用者向け画面では、金額を **千円単位** で表示する。

表示例:

- 月間目標: 5,000千円
- 月間累計: 3,750千円
- 目標まで: 1,250千円

すべての金額欄、一覧、グラフ、CSVには単位を明記する。

## 8. 入力頻度

実績は週に1回入力する。

1週間の基本区切りは次のとおり。

- 開始: 日曜日
- 終了: 土曜日

ただし、月単位で正確に集計するため、月初と月末はその月に属する日付だけを対象期間とする。

## 9. 月内期間のルール

### 例: 2026年7月

| 期間    | 対象日                       |
| ------- | ---------------------------- |
| 第1期間 | 7月1日（水）～7月4日（土）   |
| 第2期間 | 7月5日（日）～7月11日（土）  |
| 第3期間 | 7月12日（日）～7月18日（土） |
| 第4期間 | 7月19日（日）～7月25日（土） |
| 第5期間 | 7月26日（日）～7月31日（金） |

「第何週」だけでは誤認しやすいため、画面では必ず実際の日付範囲を表示する。

### 月をまたぐ場合

1週間が7月26日～8月1日の場合:

- 7月26日～7月31日を7月分に計上
- 8月1日を8月分に計上

一方の月へまとめて計上してはならない。

## 10. 週次入力項目

対象年月、対象期間、施設を選択し、次の人数を入力する。

| 訪問看護区分 | 1回訪問人数 | 2回訪問人数 | 3回訪問人数 |
| ------------ | ----------: | ----------: | ----------: |
| 医療         |        入力 |        入力 |        入力 |
| 精神科       |        入力 |        入力 |        入力 |
| 介護         |        入力 |        入力 |        入力 |

利用者が入力するのは原則として人数だけとする。

### 人数の意味

MVPでは、売上計算に使用する対象期間内の延べ人数として扱う。同じ利用者が複数日または複数期間に含まれる場合、入力元の業務集計に従って延べ計上する。

月間の重複を除いた実人数は、週次人数の単純合計からは算出しない。

## 11. 自動計算項目

- 訪問人数
- 訪問件数
- 訪問回数区分別売上
- 訪問看護区分別売上
- 施設別売上
- 期間別売上
- 月間累計売上
- 概算達成率
- 確定売上による目標達成率
- 目標までの差額
- 目標超過額
- 月末売上予測

## 12. 月間ダッシュボード

対象月について次を表示する。

- 施設別売上目標の合計、または施設ごとの内訳目標合計
- 5施設合計の概算売上
- 全施設の確定売上合計（全施設入力完了後）
- 確定売上による全体達成率
- 目標までの差額、または目標超過額
- 月末売上予測
- 入力済み期間数
- 最新期間の入力済み施設数
- 未入力の施設・期間
- 施設別達成状況
- 訪問看護区分別達成状況
- 期間別売上推移

## 13. 施設別確認

各施設について次を確認できる。

- 月間目標
- 月間累計
- 達成率
- 目標差額
- 月末予測
- 医療・精神科・介護の内訳
- 1回・2回・3回訪問の内訳
- 期間別実績
- 入力済み・未入力状況

## 14. 入力作業を減らす機能

- 保存して次の施設へ
- 前の施設へ
- 施設タブによる切り替え
- 前期間の入力内容をコピー
- すべて0件にする
- 一時保存
- 自動保存
- 入力済み表示
- 未入力施設から直接入力画面へ移動

前期間と対象日数が異なる場合は、コピー後に警告を表示する。

例:

> 前の期間は7日間、今回の期間は4日間です。コピーした人数が正しいか確認してください。

## 15. 月の締め処理

対象月の入力完了後、その月を締められるようにする。

締める前に次を確認する。

- すべての期間が入力済みか
- すべての施設が入力済みか
- 未入力欄がないか
- 異常に大きな数値がないか
- 各施設の確定売上が入力されているか
- 各施設の売上目標が入力されているか

施設別目標または確定売上が未入力の場合は警告するが、確認後の月締めは許可する。

締めた月は原則閲覧のみとする。管理者は確認操作を経て再編集できる。

## 16. 過去月

対象年月を切り替え、過去月を確認できる。

- 目標
- 概算売上
- 確定売上
- 確定売上による達成率
- 施設別売上
- 看護区分別売上
- 期間別推移
- 前月との差

## 17. データ出力・保全

- 月間集計をCSV出力できる
- データベースをバックアップできる
- バックアップから復元できる
- アップデート後も既存データが残る

## 18. 非機能要件

- オフラインで動作する
- 起動に専門知識を必要としない
- 操作画面は日本語
- 通常操作でコマンドを使わない
- 重要操作には確認画面を表示する
- 異常終了後も可能な限り入力内容を保持する
- 患者を特定する個人情報は保存しない
- 主要操作は大きなボタンで行える
- Windows 10 / 11の64bit環境で動作する
